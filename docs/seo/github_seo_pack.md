# GitHub SEO Pack

Use this checklist to improve discoverability and conversion on GitHub.

## 1) Repo About Panel

- **Description (recommended):**  
  `Safe VS Code Codex + Chrome/Edge submit auto-clicker with allowlists, dry-run mode, and visible automation status.`
- **Website:** your docs site / demo page / release page.
- **Topics:**  
  `vscode`, `codex`, `automation`, `python`, `ui-automation`, `chrome-extension`, `edge-extension`, `autoclicker`, `productivity`, `windows11`

## 2) README Conversion Elements

- One-sentence value proposition above the fold.
- "Why safe by default" section.
- Quick start with copy-paste commands.
- GIF/video demo and screenshots.
- Troubleshooting and FAQ.
- "Star this repo" CTA.

## 3) Release Strategy

- Use semantic version tags (e.g., `v1.1.0`).
- Attach zipped artifacts for desktop and extension.
- Include release notes with:
  - new features,
  - bug fixes,
  - migration/config changes.

## 4) Keyword Targets

- VS Code Codex auto submit
- submit auto clicker windows
- chrome submit auto click extension
- safe UI automation allowlist
- dry-run UI automation

## 5) Social Preview

- Add a clean OG image (1280x640) with product name + value proposition.
- Keep title readable in dark/light backgrounds.
